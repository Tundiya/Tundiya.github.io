My Review of the previous module on JavaScript drawing and animation.

4/6/22

I have recently finished the first module of the programming course here on Khan Academy and would like to jot down some thoughts about it! First off I would say that it was wildly successful! Most of the concrete learning for a first time programmer really seemed to happen in the bonus material of each challange.Although, many times I found myself stuck the sheer act of sticking with it and figuring things out, either by hitting the documentation or other students comments, really made the most progress for me. Although the tutorials and talk throughs are great, I think for the purpose of more effectively commiting these concepts to memory the struggle of trying to figure things out and just simply trying things out is huge. JavaScript itself seems to be rather intuitive, I am sure it is designed that way! Getting a good grip on for loops and nested for loops is a huge help. There is an important thing I would like to point out that took me a little while to realize. When for loops were first taught in the course it did not click in for me that loops don't have to be set against array.length, they can in fact just be set to an arbitrary amount. So as to say that if I need a loop to run 40 times, which I ended up doing, you can just set i less than 40. DOES NOT NEED TO BE LESS THAN ARRAY.LENGTH. It was a huge moment for me when I finally figured out that I can just make a for loop to push as many random numbers into an array as I needed, for example in the make it rain project.
